# 从单片机到嵌入式开发——(2) C51程序基础

date: 2020-11-28

author: 西风镭

location: 宣城，合肥工业大学宣城校区

单片机采用的是标准C语言，兼容标准C语言下的各种库函数。相比标准C语言来说扩展了三个特殊变量：特殊功能寄存器变量sfr、特殊功能寄存器位变量sbit和位标量bit。sfr变量初始需要定义为该寄存器的物理地址，sbit变量初始需要定义为该寄存器位的物理地址，bit型变量为标量，无需初始定义。sfr型变量可读写为八位二进制数，sbit型或者bit型变量读写为一位二进制数。一般寄存器均预先定义在<reg51.h>或者<reg52.h>头文件中，直接引用即可。

最常用的特殊功能寄存器即P0-P3。这四个特殊功能寄存器与P0-P3四组IO口一一对应，从而实现对IO口的读写操作。其定义为：

sfr P0    = 0x80;
sfr P1    = 0x90;
sfr P2    = 0xA0;
sfr P3    = 0xB0;

以P0口为例，其对应的8个引脚物理地址为0x80-0x87。若我们需要对P0.3引脚进行读写操作，则需要定义一个sbit变量为0x83。这种方法比较麻烦，所以我们可以使用简单的运算使得这个过程易于理解。我们知道，0与任意数字异或运算的结果为数字本身，因此我们可以通过异或运算来实现地址的转换。以P0.3引脚为例，P0的地址为0x80即10000000B，将其与3异或，即得10000011B，即0x83。因此我们可以通过定义P0^3来实现引脚定义，这种表示方法比较通俗易懂。

单片机程序的基本结构为初始化-循环，单片机所运行的程序应当是无停止的永久循环运行过程。基本结构如下：

#include <reg52.h>//引用头函数
//全局变量与函数定义
sbit port=P1^0;//定义port引脚为P1.0
void Init();//初始化函数，代指需要预先执行且仅执行一次的代码部分
void Loop();//循环函数，代指需要一直持续不断执行的部分。

void main (void)//主函数main
{
	Init();//初始化函数
	while(1)//无限循环
	{
		Loop();//执行循环函数
	}
}

void Init()
{
	P0 = 0x00;//令P0端口均为低电平
	P1 = 0x00;//令P1端口均为低电平
	P2 = 0x00;//令P2端口均为低电平
	P3 = 0x00;//令P3端口均为低电平
}

void Loop()
{
	port = 1;//令port引脚为高电平
}

这段代码的功能为将所有端口的所有引脚电平全部置低，并将P1.0引脚的电平单独置高。若将P1端口所有引脚连接LED发光二极管，另一端接地，则可以观察到仅P1.0引脚对应发光二极管点亮。

单片机最常用的函数为delay延时函数。在许多情况下，我们都需要通过一个延时函数来对单片机进行一个软件延时操作，比如LED灯的交替亮灭，数据传输的等待，或者实现某种外设装置所需要的特定时序。普通的软件延时函数一般通过循环执行空指令来进行指定时长的延时。比如下述延时5毫秒的延时函数：

void delay5ms(void)   //误差 0us
{
    unsigned char a,b;//unsigned char类型
    for(b=19;b>0;b--)
        for(a=130;a>0;a--);
}

单片机内的一般变量常使用数据类型为PC端不常用的unsigned char和unsigned int。因为单片机上的ROM空间通常是极小的，如果变量占用空间过大的话会导致ROM空间不够。
