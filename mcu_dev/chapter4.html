# 从单片机到嵌入式开发——(4) 独立与矩阵按键

date: 2020-11-29

author: 西风镭

location: 宣城，合肥工业大学宣城校区

正如电脑需要键盘鼠标显示屏这些输入输出设备，单片机也需要一些用来输入和输出的装置。此处，我们主要讲的是单片机一种十分重要的输入装置——按键。

独立按键是一种最简单的输入设备，其通常的使用方法是一端接地另一端连接引脚。当按键按下时，单片机引脚会被置低，此时读取单片机对应引脚可以得到一个低电平信号0，即判断按键按下，并运行对应程序。如图所示，按键按下时单片机引脚置低，连接在同一端口的LED灯得以点亮。

但在实际使用过程中，按键总是不那么灵敏的。可能当你没有按下按键时，单片机引脚由于电流波动误判了一个低电平，导致不应工作的按键对应程序工作了。或者单片机反复检测按键，导致当按键没有及时松开时会被误判为多次按下按键，如图所示。

该程序的功能是每按一次按键数码管计数加一。但是很明显如图并没有实现功能。因此我们必须要进行按键的去抖操作。

按键去抖的基本流程包括：检测按键->间隔->再次检测->确认按下->等待松开。程序如下：

#include <reg52.h>
sbit key = P2 ^ 0;
void delay5ms(void)   //误差 0us
{
	unsigned char a,b;
	for(b = 19; b > 0; b--)
		for(a = 130; a > 0; a--);
}

void main(void)
{ 
	while (1)
	{
		if(key == 0)
		{
			delay5ms();
			if(key == 0)
			{
				P1++;
				while(key == 0);
			}
		}  
	}
}

如图，去抖之后按键的功能得到了很好的体现。

但是，当我们需要实现的功能足够多的时候，每一个按键需要占用一个引脚就会导致引脚资源不足。在这种情况下，我们可以采用矩阵的方式，将引脚的需求量从乘法降为加法，从而实现引脚占用数量的减少。

矩阵键盘检测的基本原理与普通按键大致相同，依然是通过将高电平引脚拉低成低电平实现检测。但其与独立按键最大的区别在于另一端并不接地，而是每一个按键两端联通两个引脚。令两个引脚初始时提供不同的电平，按下按键后高电平一端会被拉低，检测高电平一端则知道该按键被按下。

矩阵键盘的扫描大致有两种办法，前者便于书写，而后者便于理解。第一种方式是按行按列各检测一次。比如说，先按行对列坐标进行检测，检测到列坐标按下后检测按列对行坐标检测。比如我们按下了位于(1,1)处的按键1，我们首先令行线对应引脚置低（如图所示P2.0、P2.1），列线对应引脚（如图所示P2.2、P2.3）置高。若位于(1,1)处的按键1被按下，则P2.2会被置低，我们可以检测出按下的按键位于第一列，但不确定是哪一行。但此时我们可以令返回值val为1。

然后我们再按列对行坐标检测。我们令列线对应引脚置低（如图所示P2.2、P2.3），行线对应引脚（如图所示P2.0、P2.1）置高。若位于(1,1)处的按键1被按下，则P2.0会被置低，我们可以检测出按下的按键位于第一行。此时我们将我们之前得到的返回值val与对应的行数1进行一个运算，令val=val+每行的按键数2*(行数1-1)，即可得到最终的结果val=1。

这部分的代码如下：

#include <reg52.h>
sbit key_h1 = P2 ^ 0;
sbit key_h2 = P2 ^ 1;
sbit key_s1 = P2 ^ 2;
sbit key_s2 = P2 ^ 3;
void delay5ms(void)   //误差 0us
{
   unsigned char a,b;
   for(b = 19; b > 0; b--)
      for(a = 130; a > 0; a--);
}

void main(void)
{
	unsigned char val = 0;
	while (1)
	{
		key_h1 = 0;
		key_h2 = 0;
		key_s1 = 1;
		key_s2 = 1;
		if(key_s1 == 0 || key_s2 == 0)
		{
			delay5ms();
			if(key_s1 == 0 || key_s2 == 0)
			{
				val = (key_s1 == 0 ? 1 : 2);
			}
		}
		key_h1 = 1;
		key_h2 = 1;
		key_s1 = 0;
		key_s2 = 0;
		if(key_h1 == 0 || key_h2 == 0)
		{
			val += (key_h1 == 0 ? 0 : 2);
			while(key_h1 == 0 || key_h2 == 0);
		}
		P1 = val;
   }
}

第二种方式是分行独立检测。在循环之中对于每一行均进行一次检测，若该行没有被按下则自动跳过，在按下行检测按下列直接得到对应结果。这种方式书写起来较为繁琐，但便于理解。

在每一行的检测过程中，我们仅将其对应的行线引脚置低，其他行线对应的引脚均置高，这样就可以保证在这个过程中检测到按下的必定是该行的按键。若该行无按键按下，则检测下一行，多次循环后即可得到结果。

使用该方式的代码如下：

#include <reg52.h>
sbit key_h1 = P2 ^ 0;
sbit key_h2 = P2 ^ 1;
sbit key_s1 = P2 ^ 2;
sbit key_s2 = P2 ^ 3;
void delay5ms(void)   //误差 0us
{
   unsigned char a,b;
   for(b = 19; b > 0; b--)
      for(a = 130; a > 0; a--);
}

void main(void)
{
	unsigned char val = 0;
	while (1)
	{
		key_h1 = 0;
		key_h2 = 1;
		key_s1 = 1;
		key_s2 = 1;
		if(key_s1 == 0 || key_s2 == 0)
		{
			delay5ms();
			if(key_s1 == 0 || key_s2 == 0)
			{
				val = (key_s1 == 0 ? 1 : 2);
				while(key_s1 == 0 || key_s2 == 0);
			}
		}
		key_h1 = 1;
		key_h2 = 0;
		key_s1 = 1;
		key_s2 = 1;
		if(key_s1 == 0 || key_s2 == 0)
		{
			delay5ms();
			if(key_s1 == 0 || key_s2 == 0)
			{
				val = (key_s1 == 0 ? 3 : 4);
				while(key_s1 == 0 || key_s2 == 0);
			}
		}
		P1 = val;
   }
}

我们最常用的矩阵键盘是4x4的16值矩阵键盘，由于其正好占用一个端口的全部八个引脚，所以我们可以对整个端口进行直接操作来实现对应结果的查找。如图是一个4x4矩阵键盘，按下对应按键可以显示对应的数字。

实现代码如下：

#include <reg52.h>
void delay5ms(void)   //误差 0us
{
   unsigned char a,b;
   for(b = 19; b > 0; b--)
      for(a = 130; a > 0; a--);
}

void main(void)
{
	unsigned char val = 0;
	while (1)
	{
		
		P2 = 0x0f;
		if(P2 != 0x0f)
		{
			delay5ms();
			if(P2 != 0x0f)
			{
				val = (P2 == 0x07 ? 3 : (P2 == 0x0b ? 2 : (P2 == 0x0d ? 1 : 0)));
			}
		}
		P2 = 0xf0;
		if(P2 != 0xf0)
		{
			val += (P2 == 0x70 ? 12 : (P2 == 0xb0 ? 8 : (P2 == 0xd0 ? 4 : 0)));
			while(P2 != 0xf0);
		}
		P1 = val;
   }
}
