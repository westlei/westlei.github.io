# 从单片机到嵌入式开发——(0) 背景知识

date: 2020-09-23

author: 西风镭

location: 宣城，合肥工业大学宣城校区

作者按: 本系列教程为一名电子信息专业本科生学习单片机与嵌入式开发过程中的一系列心得体会，于2020年疫情期间陆续完成，如有错误敬请指教。

单片机又称单片微控制器、微控制单元（MCU），它是把中央处理器(CPU)的频率与规格做适当缩减，并将内存、定时器及多种外部接口整合在单一芯片上，形成的芯片级的计算机，为不同的应用场合做不同组合控制。单片机在生活中有着广泛的应用，大到工厂机床、汽车、高铁、飞机，小到洗衣机、电冰箱、微波炉等随处可见的家用电器。在现代生活中，各种各样的单片机在生活中无处不在。因此，单片机开发也是电子、计算机、自动控制等专业在硬件领域需要掌握的基础技能之一，在学科竞赛、创新创业竞赛等方面有着十分广泛的应用。

单片机(MCU)与中央处理器(CPU)构成了现实世界与互联网世界的桥梁。相比于CPU的高性能、高功耗、高通用性，单片机主要的特点在于低功耗和专用性。在实际应用中，单片机的应用场景在于工业控制、家电和移动互联(手机等嵌入式设备)，而CPU的应用场景则集中于家用电脑、服务器和大型系统。但随着科技的进步，尤其是以高通、麒麟、联发科等一系列超低制程处理器(本质仍然是单片机)的出现，部分平板电脑和笔记本电脑也开始或计划使用单片机作为核心芯片。

目前从架构来看，单片机主要分为以8051代表的低端处理器(即俗称的51单片机)和以ARM为代表的中高端处理器。前者工艺简单，成本极低，学习较为容易，主要用于生活中常见家电的控制，也是绝大部分单片机新手入门的不二选择。后者工艺复杂，成本偏高，学习难度较大，常用于大型家电和智能设备，但实现复杂功能大多需要依靠ARM系列单片机。此外，还有一些诸如Arduino、树莓派等的变种，此处不表。由于ARM单片机广泛运用于嵌入式设备控制，一些地方将嵌入式开发狭义指代对ARM系列单片机的系统开发。

一般的单片机学习路线为从51系列单片机开始，到基于ARM架构但较为简单的ARM Cortex-M系列单片机，最后到手机等智能设备常用的ARM Cortex-A系列芯片。单片机的学习是一个软硬件结合的过程，在硬件方面需要对电路分析、模拟电路、数字电路等知识有一定的了解，而软件层面则需要精通C语言程序设计、掌握一定的汇编语言知识，并了解算法、操作系统和计算机网络等内容。

单片机开发的学习与使用主要有如下分类：

硬件层面由浅到深为：

集成开发板→单片机核心板→单片机裸片

集成开发板上载有常见的多种传感器及设备驱动装置，可按教程直接进行烧录并观察结果，对新手非常友好，但由于各硬件排布确定，在复杂设计要求下很容易出现设备冲突等现象。单片机核心板则只包含单片机程序下载和运行的基本电路，需要手动外接其他设备，相对较为自由，但开发难度同时增大。裸片则需要使用面包板或者自制PCB板来制作相应的核心电路，具有极大的自由度，但也需要相当高的电路知识水平。

软件层面由底层到顶层则是：

汇编语言编程→汇编/C语言混合编程→C语言面向硬件(寄存器)直接编程→库函数开发编程→运用操作系统开发

程序开发的底层和顶层是对于硬件而言的。对于最底层的汇编语言开发而言，所有的语句均是对单片机的硬件寄存器进行直接操作，需要对单片机的硬件部分有深刻的理解，难度相当大，但可以实现非常精确的控制。C语言面向寄存器编程和库函数开发则是最常用的两种开发手段，前者仅需要了解单片机的基本组成即可运用C语言进行操作并实现开发，而后者则通过调用硬件开发者已经写好的库函数即可进行操作。后者多见于基于ARM架构的复杂单片机系统。移植和运用操作系统进行开发可实现分时多任务并行等功能，通过调用操作系统API实现操作，需要对该操作系统有比较深刻的理解，难度同样较大，但在开发基于单片机的复杂大型程序时往往会起到事半功倍的作用。

本教程主要围绕单片机的软件开发展开，主要内容是51单片机的面向寄存器开发方式。硬件部分的相应内容会在使用仿真软件进行程序仿真运行时提及。
